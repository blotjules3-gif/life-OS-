Raisonnement interne du LLM (invisible pour l'utilisateur)
⟳ Agent loop — itération 1
1
Identification des problèmes : "stressé" → problème mental/émotionnel. "j'arrive plus à dormir" → problème de sommeil. Deux problèmes distincts à traiter.
2
Lookup dans PROBLEMES_SOLUTIONS : MENTAL → "je suis stressé / sous pression" trouvé. Solutions : [fitness], [mind], [productivity], [finance]. SOMMEIL → "je n'arrive pas à m'endormir" trouvé. Solutions : [sleep], [mind], [nutrition].
3
Cross-référencement avec les modules actifs : Modules actifs = fitness, nutrition. Modules manquants = mind, sleep, productivity, finance.
4
Décision selon le protocole : Commencer par les modules déjà actifs → optimiser fitness (sport réduit cortisol) + nutrition (arrêter café après 14h). Puis proposer les modules manquants qui couvrent le mieux les deux problèmes → mind couvre stress ET sommeil. sleep couvre directement le problème de nuit.
5
Plan d'action : 1 optimisation config fitness → 1 optimisation config nutrition → 2 add_module (mind + sleep) → créer 1 habitude rapide → répondre en 2 phrases max.
Règle clé appliquée : Ne jamais proposer add_module pour un module déjà actif. Fitness et nutrition sont actifs → le LLM propose de les optimiser, pas de les rajouter.
